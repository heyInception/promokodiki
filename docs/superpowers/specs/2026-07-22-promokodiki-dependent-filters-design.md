# Зависимые фильтры и индикатор загрузки Promokodiki

## Цель

Доработать Promokodiki AJAX Filter так, чтобы на главной категории и бренды предлагали только совместимые комбинации, контекстные dropdown скрывались при отсутствии реального выбора, а любая AJAX-загрузка имела заметный индикатор прогресса.

Изменения применяются ко всем категориям и всем страницам магазинов. Страница AliExpress используется только как воспроизводимый пример ошибки.

## Поведение главной страницы

Категория и бренд остаются одиночными dropdown и фильтруют промокоды по пересечению `AND`.

- Без выбранных значений выводятся все категории и бренды, содержащие активные промокоды.
- После выбора категории список брендов содержит только бренды с активными промокодами в этой категории.
- После выбора бренда список категорий содержит только категории с активными промокодами этого бренда.
- Если бренд выбран первым, последующий выбор категории не сбрасывает бренд: доступные категории уже ограничены совместимыми вариантами.
- Очистка категории расширяет список брендов до полного, сохраняя выбранный бренд; очистка бренда аналогично расширяет список категорий, сохраняя выбранную категорию.
- Режим «Популярное за неделю» остаётся эксклюзивным и очищает категорию, бренд и сортировку.
- При загрузке старого URL с несовместимой парой категория имеет приоритет, несовместимый бренд сбрасывается.

Списки обновляются в том же AJAX-ответе, что и карточки. Отдельного запроса только для dropdown не будет.

## Страница категории

Контекст остаётся ограничен текущей категорией и её потомками.

- Если у текущей категории нет подкатегорий, dropdown категории не рендерится.
- Если подкатегории есть, dropdown содержит текущую категорию и всех потомков с сохранением иерархических подписей.
- Сортировка отображается всегда.

## Страница магазина

Брендами считаются все термины `shops_category`, связанные с активными промокодами текущего магазина. Они не ограничиваются дочерними терминами текущего магазина: именно это ограничение вызывало ошибку, при которой на AliExpress и других страницах отображался один вариант.

- Если найден ноль или один связанный бренд, dropdown брендов не рендерится.
- Если найдено два и более связанных бренда, dropdown показывает их все.
- Выбор бренда остаётся ограничен текущей страницей магазина и применяется вместе с сортировкой.
- Правило одинаково для всех магазинов.

## Серверный поток данных

Контекст хранит полный безопасный набор допустимых ID для текущего экрана. Дополнительно сервис вариантов строит доступные dropdown на основании текущего состояния:

- категории зависят от выбранного бренда;
- бренды зависят от выбранной категории;
- учитываются только опубликованные и активные промокоды;
- истёкшие предложения не создают доступный вариант, даже если настройка разрешает показывать их в конце выдачи.

Первичный GET-рендер и AJAX используют одну и ту же функцию построения вариантов. AJAX-ответ дополняется нормализованным состоянием и массивами `category_options` и `brand_options`, чтобы клиент не повторял бизнес-логику WordPress.

## Клиентское поведение

JavaScript после успешного ответа:

1. заменяет карточки или добавляет следующую порцию;
2. для обычной фильтрации обновляет варианты обоих dropdown;
3. сохраняет выбранные совместимые значения;
4. обновляет URL и историю нормализованным состоянием сервера.

При «Показать ещё» варианты dropdown не перестраиваются, чтобы не сбивать текущий интерфейс.

## Лоадер

Во время любого запроса фильтра или «Показать ещё»:

- над областью результатов отображается визуальный spinner;
- контейнер получает `aria-busy="true"`;
- элементы формы и кнопка «Показать ещё» временно блокируются;
- повторный выбор отменяет предыдущий запрос через `AbortController`;
- после успеха, ошибки или отмены spinner скрывается;
- при ошибке сохраняется существующая кнопка повторного запроса.

Лоадер не удаляет текущие карточки и не меняет высоту выдачи резким пустым состоянием.

## Кэширование и безопасность

- Полные контексты и вычисленные совместимые варианты используют версионированные transient-ключи.
- Версия кэша обновляется при создании/редактировании терминов, изменении связей промокода и сохранении промокода.
- AJAX сохраняет nonce-проверку, нормализацию ID и серверную проверку допустимых терминов.
- HTML и подписи экранируются WordPress-функциями.

## Тестирование

Регрессионные тесты должны подтвердить:

- категория сужает бренды;
- бренд сужает категории и сохраняется после совместимого выбора категории;
- несовместимая пара из GET нормализуется с приоритетом категории;
- категория без потомков не выводит dropdown;
- категория с потомками выводит dropdown;
- магазин с одним связанным брендом скрывает dropdown;
- магазин с несколькими связанными брендами показывает все варианты, включая термины вне ветки текущего магазина;
- AJAX возвращает обновлённые варианты и нормализованное состояние;
- JavaScript перестраивает dropdown и управляет лоадером;
- «Показать ещё», сортировки, недельная популярность, URL/history и подсчёт кликов не регрессируют.

После автоматических тестов выполняется браузерная проверка главной, категории без потомков, категории с потомками, магазина с одним брендом и магазина с несколькими брендами.
