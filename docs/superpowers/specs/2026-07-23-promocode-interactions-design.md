# Интерактивные промокоды — дизайн

## Цель

Сделать карточки промокодов единообразными, кликабельными и измеряемыми: посетитель получает код в модальном окне, переход в магазин учитывается как применение, а статистика, статусы и реакции управляются из одного плагина.

## Границы ответственности

`promokodiki-ajax-filter` владеет настройками, серверными правилами, хранилищем применений/реакций и AJAX API. Тема владеет единым шаблоном карточки, разметкой модального окна, стилями и браузерным клиентом. Удаляем из темы старые обработчики лайков, счётчиков и дублирующую разметку карточек.

## Данные и правила

- Badge вычисляется на сервере и ровно один: `expired`, затем `new`, затем `popular`.
- `expired` означает, что `_promocode_expiry_date` раньше текущей даты WordPress.
- `new` означает публикацию не старше настраиваемого числа дней; по умолчанию 14.
- `popular` использует ту же недельную сумму таблицы `promokodiki_click_stats`, что и режим `paf_popular=1`; минимальный порог настраивается.
- Применение регистрируется только при подтверждённом переходе из модального окна. Оно увеличивает `_promocode_used_count` и дневную запись статистики, если анонимный посетитель не был учтён для этого промокода в течение заданного периода. Период по умолчанию 24 часа; `0` учитывает каждый клик.
- Реакция посетителя хранится как одна сменяемая запись `like` или `dislike` для промокода и анонимного browser ID. Агрегаты остаются в `_promocode_likes` и `_promocode_dislikes`.

## Настройки

В существующем экране «Промокоды → AJAX-фильтр» добавляется секция «Интерактивные промокоды»:

- срок badge «Новое» в днях (14);
- период антидублей применения в часах (24, `0` разрешает повторный учёт);
- минимальное число применений за окно `popular_days` для badge «Популярное»;
- режим истекших карточек: оставить действия активными или отключить действия.

Все значения проходят строгую нормализацию и имеют безопасные значения по умолчанию.

## Карточка и модальное окно

Единый template part принимает ID промокода и формирует бренд через `get_term_link()`, поэтому имя и логотип бренда ведут на его архив во всех контекстах. Кнопки «Посмотреть код» и «Перейти в магазин» не уходят напрямую: открывают одну модалку с контекстом карточки. Первая сразу отображает и копирует код; вторая фокусирует кнопку перехода. Акция без кода показывает только сценарий перехода. Подтверждённый переход сначала вызывает AJAX учёта, затем безопасно открывает сохранённую URL магазина в новой вкладке.

## Надёжность и безопасность

AJAX требует nonce, существующий опубликованный `promocode` и разрешённое значение действия. Внешняя URL берётся с сервера из метаданных, а не из клиентского запроса. Анонимный ID — случайное значение в first-party cookie; его хэш хранится в собственной таблице без персональных данных. Операции обновляют агрегаты после транзакционно безопасной смены голоса/применения.

## Проверки

PHP WP-CLI тесты покрывают нормализацию настроек, правила badge, дедупликацию применений, смену реакции и ответ AJAX. JS-тесты покрывают перенос данных карточки в модалку, копирование и задержку внешнего перехода. Проверяются PHP syntax, существующий набор тестов и PHPCS.
