# Promokodiki AJAX Filter — дизайн плагина

Дата: 2026-07-22

## Цель

Заменить Filter Everything отдельным плагином для фильтрации промокодов без перезагрузки страницы. Плагин должен учитывать контекст страницы, управляться из панели WordPress, сохранять состояние в URL и иметь рабочий GET-режим без JavaScript.

## Область работ

Плагин работает только в следующих местах:

- главная страница с ACF-секцией `promokody`;
- страницы терминов `promocode_category`;
- страницы терминов `shops_category`.

Архив `/promocodes/`, который сейчас служит каталогом категорий, остаётся без изменений. Существующая тема `promokodiki` продолжает отвечать за карточки и визуальное оформление блока `.promocodes__filters`.

## Функциональные требования

### Главная страница

Блок содержит:

- одиночный dropdown «Категории» со всеми доступными терминами `promocode_category`;
- одиночный dropdown «Бренды» со всеми доступными терминами `promocode_brand`;
- переключатель «Популярное за неделю»;
- dropdown сортировки.

Категория и бренд могут быть выбраны одновременно. В этом случае запрос использует условие «категория И бренд».

Включение «Популярного за неделю»:

- очищает выбранную категорию;
- очищает выбранный бренд;
- сбрасывает обычную сортировку;
- сортирует по сумме кликов за последние семь календарных дней по убыванию.

### Страница категории промокодов

Блок содержит:

- один dropdown с текущей категорией и всеми её дочерними категориями любого уровня;
- dropdown сортировки.

Выбор одиночный. Текущая категория означает всю текущую ветку. Дочерние уровни отображаются в dropdown с визуальными отступами, но без отдельного дерева или мультивыбора.

### Страница магазина

Блок содержит:

- один dropdown «Бренды» только с брендами, у которых есть активные промокоды в текущем термине `shops_category`;
- dropdown сортировки.

Бренды без активных промокодов не показываются. Выбор одиночный, значение по умолчанию — «Все бренды».

### Сортировки

Доступны четыре варианта:

1. «Сначала новые» — `post_date DESC`, вариант по умолчанию.
2. «Сначала популярные» — `_promocode_used_count DESC`.
3. «Скоро истекают» — ближайшая будущая дата `_promocode_expiry_date` первой; записи без даты идут после датированных.
4. «Сначала старые» — `post_date ASC`.

«Популярное за неделю» является отдельным взаимоисключающим режимом, а не пятой обычной сортировкой.

### Истёкшие промокоды

По умолчанию истёкшие промокоды исключаются. Администратор может включить их показ в конце выдачи. Когда настройка включена, активные записи всегда располагаются раньше истёкших, независимо от выбранной сортировки.

### Пагинация

Первая порция заменяет текущий список. Кнопка «Показать ещё» загружает следующую порцию и добавляет карточки в конец. Нумерованной AJAX-пагинации не будет.

## Состояние и URL

Фильтры являются обычной GET-формой и имеют progressive enhancement:

- без JavaScript форма отправляется с перезагрузкой;
- с JavaScript запрос выполняется в фоне;
- выбранные значения сохраняются в GET-параметрах;
- состояние можно открыть по прямой ссылке;
- обновление страницы и кнопки браузера «Назад»/«Вперёд» восстанавливают состояние.

Имена параметров должны быть изолированы префиксом плагина, чтобы не конфликтовать с Filter Everything и другими расширениями.

## Архитектура

Рабочее название плагина: `Promokodiki AJAX Filter`.

Компоненты плагина:

1. Bootstrap-класс регистрирует зависимости, ресурсы, shortcode/функцию вывода и хуки жизненного цикла.
2. Context Resolver определяет главную страницу, текущий `promocode_category` или текущий `shops_category` и возвращает разрешённые фильтры.
3. Query Builder преобразует нормализованное состояние в единый `WP_Query`. Этот сервис используется и GET-режимом, и AJAX.
4. Filter Renderer выводит доступные текущему контексту элементы внутри существующей структуры `.promocodes__filters`.
5. Results Renderer повторно использует `template-parts/promocode-card.php` текущей темы и предоставляет фильтр-хук для замены рендера.
6. AJAX Controller обрабатывает замену результатов, «Показать ещё» и клики по промокодам.
7. Click Statistics Repository хранит дневные агрегаты.
8. Settings Page работает через WordPress Settings API.
9. Compatibility Notice предупреждает администратора, если активна бесплатная или Pro-версия Filter Everything.

В теме шорткоды `[fe_widget]` и `[fe_sort]` в целевых блоках заменяются одним вызовом плагина. Бизнес-логика фильтра в тему не переносится.

## Поток данных

1. Сервер формирует начальную GET-выдачу и доступные варианты фильтра.
2. JavaScript перехватывает изменение dropdown или переключателя, нормализует состояние и обновляет URL через History API.
3. Предыдущий незавершённый запрос отменяется.
4. AJAX Controller повторно проверяет nonce, контекст и допустимые значения.
5. Query Builder создаёт запрос, Results Renderer возвращает HTML карточек и метаданные пагинации.
6. При обычной фильтрации HTML заменяет список; при «Показать ещё» — добавляется в конец.
7. Ошибка не очищает текущий список: пользователь получает сообщение и кнопку повторного запроса.

## Статистика кликов

Каждый клик по «Посмотреть код» или «Перейти в магазин» считается отдельным применением. Регистрация и дедупликация по браузеру не используются.

Клик атомарно увеличивает:

- существующий общий счётчик `_promocode_used_count`;
- дневной агрегат в таблице плагина.

Таблица хранит:

- `promocode_id`;
- календарную дату в часовом поясе WordPress;
- количество кликов;
- уникальный ключ по `promocode_id + date`;
- индексы по дате и ID промокода.

Недельная популярность начинает собираться после активации плагина. Исторические данные не восстанавливаются и не подменяются общим счётчиком. Если за последние семь дней нет кликов, выводится «Данных пока нет».

## Настройки администратора

Раздел «Промокоды → AJAX-фильтр» доступен только пользователям с `manage_options` и содержит:

- число карточек в первой порции;
- число карточек для «Показать ещё»;
- сортировку по умолчанию;
- список включённых вариантов сортировки;
- период популярности, по умолчанию 7 дней;
- переключатель показа истёкших в конце;
- подписи dropdown, режима популярности и кнопок.

Значения проходят серверную валидацию. Нельзя сохранить пустой набор сортировок или сортировку по умолчанию, отсутствующую в разрешённом наборе.

## Безопасность

- AJAX-запросы защищаются nonce.
- Все входные данные очищаются и проходят проверку по белым спискам.
- Выбранный термин должен принадлежать разрешённому набору текущего контекста.
- Для счётчика принимаются только опубликованные записи `promocode`.
- Вывод экранируется в соответствии с типом данных.
- Управление настройками требует `manage_options`.
- Для пользовательских SQL-операций применяются подготовленные запросы `$wpdb->prepare()`.

## Производительность

- Недельная статистика хранится дневными агрегатами, а не отдельными событиями.
- Списки терминов dropdown кэшируются и инвалидируются при изменении терминов или связей записей.
- HTML результатов не получает долгий прикладной кэш, чтобы статистика и сроки действия не устаревали.
- Запросы получают стабильную вторичную сортировку по ID, исключающую дубли между порциями.
- Реализация должна работать в локальном SQLite WordPress Studio и на production MySQL/MariaDB через стандартные WordPress API.

## Интерфейс и доступность

Используется уже свёрстанный дизайн `.promocodes__filters`. На главной визуально остаются «Категории», «Бренды», «Популярное за неделю» и сортировка; пункт «Проверенные» не реализуется.

Сохраняется текущее адаптивное поведение темы. Дополнительно:

- все поля получают явные label;
- управление работает с клавиатуры;
- состояние загрузки обозначается визуально и через `aria-busy`;
- изменение результатов объявляется через `aria-live`;
- во время запроса предотвращаются повторные конфликтующие действия;
- при пустой выдаче показывается контекстное сообщение.

## Совместимость и миграция

- Filter Everything и Filter Everything Pro не удаляются и не деактивируются автоматически.
- Плагин показывает предупреждение об их активности.
- После локальной проверки обе версии деактивируются вручную, сохраняя настройки для отката.
- Существующие карточки, модальные окна, лайки и ссылки промокодов продолжают использовать код темы.
- Архив `/promocodes/` и остальные страницы сайта не меняются.

## Проверка

Нужно проверить:

- PHP-синтаксис и WordPress Coding Standards;
- активацию, создание таблицы и повторную безопасную активацию;
- права и валидацию настроек;
- одинаковые результаты GET и AJAX;
- главную с категорией, брендом, их комбинацией и недельным режимом;
- категорию с текущим термином и дочерними уровнями;
- магазин только с допустимыми брендами;
- все четыре сортировки;
- исключение и вывод в конце истёкших записей;
- первую порцию и «Показать ещё» без дублей;
- прямые URL и навигацию «Назад»/«Вперёд»;
- учёт каждого клика в общем и дневном счётчиках;
- нулевую недельную статистику;
- отмену устаревших запросов, ошибку и повтор;
- клавиатуру, ARIA и мобильную вёрстку;
- отсутствие конфликтующего вывода после деактивации Filter Everything.

- рабочий режим без JavaScript;
